 Thank you for taking the time to contribute!\hypertarget{md_CONTRIBUTING_autotoc_md15}{}\doxysection{\texorpdfstring{Reporting bug}{Reporting bug}}\label{md_CONTRIBUTING_autotoc_md15}

\begin{DoxyItemize}
\item Please read the documentation, maybe it\textquotesingle{}s a limitation, a known issue of that release. This might also help to clear false expectation from the library/tool itself, or help you to classify your request not as a bug, but rather as an enhancement.
\item Ensure that the bug was not already reported by searching on Git\+Hub under \href{https://github.com/khushal-banks/logbank_c/issues}{\texttt{ Issues}}.
\item If you have not found an open issue addressing the problem, open a new one. Be sure to include a title and clear description, with as much relevant information as possible.
\item If possible attach the screenshot of current output and write about the expected output in detail.
\end{DoxyItemize}\hypertarget{md_CONTRIBUTING_autotoc_md16}{}\doxysection{\texorpdfstring{Suggesting enhancements}{Suggesting enhancements}}\label{md_CONTRIBUTING_autotoc_md16}

\begin{DoxyItemize}
\item Enhancement suggestions are tracked as Git\+Hub issues.
\item Use a clear and descriptive title for the issue to identify the suggestion.
\item Describe the current behavior and explain which behavior you expected to see instead and why.
\end{DoxyItemize}\hypertarget{md_CONTRIBUTING_autotoc_md17}{}\doxysection{\texorpdfstring{Pull Requests}{Pull Requests}}\label{md_CONTRIBUTING_autotoc_md17}

\begin{DoxyItemize}
\item Open a new Git\+Hub pull request with the patch. Check, \href{https://github.com/firstcontributions/first-contributions}{\texttt{ first-\/contributions}} for details.
\item Ensure the PR description clearly describes the problem and solution. Include the relevant issue number if applicable.
\item Think about test-\/ability. Please write test(s) for the problem you have fixed. Make sure that existing tests do not get broken. 
\end{DoxyItemize}
