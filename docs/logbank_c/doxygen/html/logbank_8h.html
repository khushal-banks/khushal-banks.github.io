◆ LOG_LOCATION

#define LOG_LOCATION ( level)    (level == LOG_DEBUG || level < log_get_default_level())

This macro controls when to log location parameters for any log-request.

Location parameters are printed when requested log-level is below LOG_LEVEL (default value of log-level) Also, Location parameters are always logged for LOG_DEBUG priority (which is the minimum value for log_level)

Note
Location parameters will only be logged if, someone explicity set log_level to be below default value other wise log-request will be ignored because of lower-priority.

This is the main logging function of this library.

This function parses arguments similar to that of printf So, format argument of this function can be replaced by only argument used for log-message

Parameters

[in]locatelocation parameters for this log-request

[in]...optional arguments for log-message
Returns
>0 Success: Message logged and Positive returned value is the length of logged message
=0 Failure: Message ignored because of lower-priority
<0 Failure: Overflow occured. Negative returned value is the required length to log input log-request
