◆ LOG_BARE_MINIMUM_DATA

#define LOG_BARE_MINIMUM_DATA   (23 + (4 * 4) + 8 + LOG_MAX_FUNC_NAME_LEN + LOG_MAX_FILE_NAME_LEN)

Used for length validation of LOG_MAX_DATA_LEN.

  • 23 bytes are required by sample TRACE Header
    • 4 colors of length 4 bytes are used (run-time configurable)
    • 8 is length of "Overflow" message
    • Adding both function and file name length
Note
For easy calculation: 50 + FUNC_LEN + FILE_LEN - 3

  • 0: LOG_DEBUG

◆ LOG_MAX_DATA_LEN

#define LOG_MAX_DATA_LEN   150

Maximum allowed length for log-message.

  • Min required value ranges around (70-150) depending on value of func length and file length
    • By default: length 117 can only print overflow, so try 220
    • If you are working with longer strings in general, try 270
    • If you are using both stamps consider further increasing the limit to 285
Note
Modifying this beyond 285 will require increasing length of test messages in test02_overflow.c
